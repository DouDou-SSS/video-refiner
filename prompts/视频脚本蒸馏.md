# 视频脚本结构蒸馏 Prompt

> 用于分析博主视频的整体脚本结构、信息密度、叙事逻辑等

---

## 任务

你是一个短视频脚本分析专家。请仔细分析以下视频视频的完整内容（画面、旁白、字幕、节奏），深度蒸馏该博主的视频脚本创作方法论。

## 分析维度

### 1. 脚本整体架构
- **时长分布**：视频总时长？各部分（开场/正文/结尾）分别占多少秒？
- **段落数量**：视频分几个核心段落？
- **信息密度**：每个段落传递几个核心观点？每分钟几个知识点？

### 2. 开场结构（前5秒）
- **第一帧设计**：画面是什么？文字是什么？
- **第一句话**：旁白/字幕的第一句话是什么类型？
- **悬念制造**：怎么让观众愿意继续看下去？

### 3. 正文叙事逻辑
- **信息排列顺序**：按重要性降序？按时间线？按逻辑递进？
- **论据类型**：用数据支撑？用案例说明？用对比论证？
- **过渡手法**：从一个知识点到下一个，用什么方式衔接？
- **举例方式**：举什么类型的例子？（个人经历/行业案例/比喻类比）

### 4. 脚本节奏控制
- **快慢交替**：有没有刻意的节奏变化？
- **重点强调**：用什么方式强调核心观点？（重复？停顿？视觉强化？）
- **注意力维持**：每多少秒给一个新刺激？

### 5. 结尾设计
- **收尾方式**：总结提炼？行动号召？留下悬念？金句结尾？
- **互动引导**：怎么引导点赞、评论、关注？
- **记忆点**：最后留给观众什么印象？

## 输出要求

请按照以下结构化格式输出：

```markdown
# 博主脚本结构分析报告

## 一句话总结
（20字以内概括该博主的脚本结构特点）

## 标准脚本模板
```
[时长]：XX秒
[0-5s] 开场：...
[5-XXs] 正文段落1：...
[XX-XXs] 正文段落2：...
...
[结尾Xs] 收尾：...
```

## 叙事逻辑
- ...

## 信息密度
- ...

## 重点强调手法
- ...

## 过渡衔接方式
- ...

## 可复用脚本模板
（生成1-2个完整的脚本模板，包含时间轴）
```

## 注意事项
- 要具体到秒级别的节奏分析
- 每个判断都要有视频中的具体例子支撑
- 提炼的模板要能直接套用
- 重点关注结构规律，而非具体内容
